WORKFLOW · ENTER

Rooms Have Events. Events Have Steps. Steps Run When Events Trigger.

Every room has lifecycle events: Enter, Exit, Back, Reset, Unload. When the user arrives at a room, its Enter event fires and every step in that workflow runs in order, top to bottom.

Steps are actions — small, focused operations like setting a field, reading from inventory, or navigating to another room. Add steps from the action library on the left, then drag the ⠿ handle to reorder them.

Use Basic mode for simple configuration, Configure for all params, and JSON to edit the raw step object.
